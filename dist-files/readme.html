b1gMail Readme

Thank you very much for purchasing b1gMail! In this package, you can find all files you need to install b1gMail onto your webspace.
This file contains summarized instructions how to set up b1gMail and how to update b1gMail from further versions to the current version. More detailed instructions are contained in the b1gMail manual - this readme file serves as a quick start introduction only.

New / first installation

If you are planning a new / first installation of b1gMail, please follow the following instructions.
  1. Set up a new POP3 account an configure your CatchAll feature to redirect all incoming mails for the domains you would like to use with b1gMail to this POP3 account. (This step is not required in case you plan to install and use b1gMailServer for email receiving.)
  2. Upload all files and folders contained in the folder "upload" of this package to your webspace (e.g. using a FTP client).
  3. Provide the following files and folders at your webspace with writing permissions (normally chmod 666 for files and chmod 777 for folders):
    • serverlib/config.inc.php
    • serverlib/version.inc.php
    • admin/templates/cache/
    • languages/
    • logs/
    • plugins/
    • plugins/templates/
    • plugins/templates/images/
    • temp/
    • temp/session/
    • temp/cache/
    • templates/modern/cache/
    • data/
  4. Open the set up wizard of b1gMail by visiting the folder /setup/ in your browser (e.g. http://www.your-domain.xy/b1gmail/setup/) and follow the on-screen instructions.
  5. After installing b1gMail successfully, necessarily delete the folder "setup" from your webspace.

Update from a previous b1gMail7 version

If you are using a previous version of b1gMail7, please use the appropriate update package instead of the complete package and follow the following instructions.
Always create a full back-up (database and files) before updating!
  1. In case you update from a b1gMail version older than 7.4.0-Beta1, log in to the administrator control pabel, go to "Preferences" » "Email" and ensure mails are storen in the file system (rather then in the database). Afterwards, even if this has already been the case, go to "Tools" » "Optimize" » "File system" and run the feature to move mails from database to file system. This is required to ensure that there are not more emails left in the database. In case this step is not done as described, the updater might deny the update later!
  2. Enable the maintenance mode (using the administrator control panel at "Preferences" » "Common").
  3. Upload all files and folders contained in the folder "upload" of the update package to your webspace (e.g. using a FTP client). Overwrite existing files.
  4. Provide the following files and folders at your webspace with writing permissions (normally chmod 666 for files and chmod 777 for folders):
    • serverlib/config.inc.php
    • serverlib/version.inc.php
    • admin/templates/cache/
    • languages/
    • logs/
    • plugins/
    • plugins/css/
    • plugins/js/
    • plugins/templates/
    • plugins/templates/images/
    • temp/
    • temp/session/
    • temp/cache/
    • templates/modern/cache/
    • data/
  5. Open the update wizard of b1gMail by visiting the file /setup/update.php in your browser (e.g. http://www.your-domain.xy/b1gmail/setup/update.php) and follow the on-screen instructions.
  6. Optional: If you do not use b1gMail in UTF-8 mode but want to switch to UTF-8 mode, visit the file /setup/utf8convert.php in your web browser and follow the instructions of the conversion wizard.
  7. After updating b1gMail successfully, necessarily delete the folder "setup" from your webspace.
  8. If you are using b1gMailServer, please restart the bms-queue service.
  • Starting with b1gMail 7.3.0-Beta1, administration login requires an username. When updating from older versions, the default administrator username is "admin"

Update from b1gMail6

If you are using b1gMail6, please update to b1gMail 7.2.0 and then install the update from to this version.
